# Custom Rules

Place your custom rules in this directory. Files here will not be overwritten by upgrades.

## File Format

Each rule file should be a markdown file with:
- Clear title
- Specific rules or guidelines
- Examples where helpful

## Naming Convention

Use descriptive kebab-case names:
- `my-project-conventions.md`
- `team-coding-standards.md`
- `api-guidelines.md`
