# Skill: brainstorming

## Purpose

Generate broad, high-quality solution ideas and surface constraints before committing to a plan.

## When to Use

- Early discovery or ambiguous requests
- Need multiple options before selecting a direction
- Before writing a plan or proposal

## Steps

1. **Capture context**: Summarize goal, constraints, timeline, and existing assets
2. **Idea burst**: List diverse approaches (architecture, tooling, process), aim for 5–10 options
3. **Expand top candidates**: Add pros/cons, risks, effort, prerequisites
4. **Converge**: Pick 1–2 options; note why they win and what to validate next
5. **Outputs**: Concise options table + recommended direction + immediate next checks

## Output Format

```markdown
## Context
[Goal and constraints summary]

## Options

| Option | Description | Pros | Cons | Effort |
|--------|-------------|------|------|--------|
| A      | ...         | ...  | ...  | Low    |
| B      | ...         | ...  | ...  | Medium |

## Recommendation
[Selected option with rationale]

## Next Steps
[Validation items]
```
